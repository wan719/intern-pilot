# InternPilot 可行性分析报告

## 1. 项目名称

InternPilot：面向大学生的 AI 实习投递与简历优化平台

## 2. 项目背景

随着高校学生实习竞争加剧，大学生在投递实习岗位时常常面临简历内容不够精准、岗位要求理解不足、投递过程缺乏管理、面试准备不系统等问题。传统简历修改方式依赖人工经验，效率较低，且难以根据不同岗位 JD 进行个性化调整。

InternPilot 面向大学生实习求职场景，结合 Spring Boot 后端工程能力与大语言模型分析能力，为用户提供简历上传解析、岗位 JD 管理、AI 匹配度分析、简历优化建议和投递流程跟踪等功能，帮助学生提高岗位匹配度和投递效率。

## 3. 建设必要性

### 3.1 用户需求明确

大学生在求职过程中普遍存在以下问题：

1. 不知道自己的简历是否匹配岗位；
2. 不知道简历项目经历该如何优化；
3. 同一份简历投递所有岗位，缺乏针对性；
4. 投递记录分散，容易遗漏笔试、面试时间；
5. 面试前缺少针对岗位的复习建议。

因此，构建一个面向大学生的 AI 实习投递助手具有较强的实际价值。

### 3.2 技术实现可行

本项目主要由用户系统、简历管理、岗位 JD 管理、AI 分析、投递记录管理等模块组成。核心技术包括 Spring Boot、Spring Security、JWT、MySQL、Redis、文件解析和大语言模型 API 调用，均属于成熟技术方案，技术风险可控。

### 3.3 项目周期可控

项目可以分阶段开发：

1. 第一阶段完成用户登录、简历上传、岗位管理；
2. 第二阶段完成 AI 匹配分析和投递记录；
3. 第三阶段完成 Redis 缓存、WebSocket、Docker 部署等增强功能。

每个阶段都有明确交付物，适合个人独立开发和持续迭代。

## 4. 技术可行性

### 4.1 后端技术可行性

Spring Boot 具备成熟的 Web 开发生态，可以快速构建 RESTful API；Spring Security + JWT 可以实现前后端分离场景下的认证授权；MyBatis Plus 可以提升数据库操作效率；MySQL 能够满足用户、简历、岗位和投递记录等结构化数据存储需求。

### 4.2 AI 分析可行性

本项目通过调用大语言模型 API 完成简历与岗位 JD 的语义分析。系统将简历文本和岗位描述拼接为结构化 Prompt，并要求模型返回 JSON 格式结果。通过 Prompt 模板、异常处理和结果校验，可以保证 AI 分析结果的可用性。

### 4.3 文件解析可行性

PDF 简历可使用 Apache PDFBox 解析，DOCX 简历可使用 Apache POI 解析。解析后的纯文本存储在数据库中，供 AI 分析模块调用。

### 4.4 缓存可行性

AI 分析接口存在响应时间较长和调用成本较高的问题。系统可以使用 Redis 对相同简历与岗位组合的分析结果进行缓存，从而减少重复调用，提高响应速度。

## 5. 经济可行性

项目开发主要依赖开源框架和个人开发环境，成本较低。可能产生的成本主要包括：

1. 大语言模型 API 调用费用；
2. 云服务器部署费用；
3. 域名或对象存储费用。

在开发测试阶段，可以通过限制调用次数、使用缓存、使用免费额度等方式控制成本。

## 6. 操作可行性

系统面向大学生用户，功能流程清晰：

1. 上传简历；
2. 输入岗位 JD；
3. 查看 AI 分析结果；
4. 根据建议修改简历；
5. 保存投递记录。

操作门槛较低，适合目标用户使用。

## 7. 风险分析

### 7.1 AI 输出不稳定

风险：大模型可能返回格式不统一或内容不准确。

应对措施：

1. 使用固定 Prompt 模板；
2. 要求模型返回 JSON；
3. 对返回结果进行格式校验；
4. 保存原始返回内容，便于排查问题。

### 7.2 文件解析不准确

风险：PDF 或 DOCX 简历格式复杂，可能导致解析结果混乱。

应对措施：

1. 第一版优先支持标准文本型 PDF 和 DOCX；
2. 提供用户手动编辑解析文本的功能；
3. 后续接入 OCR 或更专业的文档解析服务。

### 7.3 API 调用成本较高

风险：频繁调用 AI API 可能增加成本。

应对措施：

1. 使用 Redis 缓存分析结果；
2. 限制普通用户调用次数；
3. 对重复请求进行去重处理。

### 7.4 项目范围过大

风险：如果一开始加入 RAG、岗位推荐、多模型切换等高级功能，可能导致项目延期。

应对措施：

1. 第一版只完成 MVP；
2. 高级功能放到第二阶段；
3. 保证核心流程优先闭环。

## 8. 结论

InternPilot 项目需求明确、技术成熟、开发周期可控、应用场景真实，适合作为大学生 Java 后端与 AI 应用结合的综合实践项目。该项目不仅能够解决大学生实习投递过程中的实际问题，也能够展示开发者在后端架构、权限认证、文件处理、缓存优化和 AI 应用集成方面的综合能力。
