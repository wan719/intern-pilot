# InternPilot 项目规划企划

## 1. 项目总体定位

InternPilot 是一个面向大学生实习求职场景的 AI 应用平台。项目以 Spring Boot 后端工程为主体，结合大语言模型能力，实现简历解析、岗位匹配、简历优化建议和投递流程管理。

项目既关注真实用户需求，也关注工程实践价值，目标是打造一个适合 GitHub 展示、简历包装和实习面试讲解的完整项目。

## 2. 项目开发原则

本项目遵循以下开发原则：

1. 先完成核心闭环，再扩展高级功能；
2. 后端优先，前端满足基本展示即可；
3. 每个模块都要有明确业务价值；
4. 技术选型优先考虑实习岗位匹配度；
5. 所有功能都要能在面试中讲清楚；
6. 文档、接口、数据库设计同步推进。

## 3. 项目阶段规划

## 第一阶段：MVP 核心功能

周期：第 1 周 - 第 2 周

目标：完成系统核心业务闭环。

主要任务：

1. 初始化 Spring Boot 后端项目；
2. 配置 MySQL、Redis、Swagger；
3. 实现用户注册登录和 JWT 鉴权；
4. 实现简历上传和文本解析；
5. 实现岗位 JD 管理；
6. 实现 AI 匹配分析；
7. 实现投递记录管理。

阶段成果：

1. 用户可以登录系统；
2. 用户可以上传简历；
3. 用户可以录入岗位 JD；
4. 系统可以生成 AI 分析结果；
5. 用户可以保存投递记录。

## 第二阶段：工程化增强

周期：第 3 周

目标：提升项目工程质量和面试亮点。

主要任务：

1. Redis 缓存 AI 分析结果；
2. 增加统一异常处理；
3. 增加参数校验；
4. 增加操作日志；
5. 完善 Swagger 接口文档；
6. 编写核心接口测试；
7. 编写 Docker Compose 配置。

阶段成果：

1. 项目具备更好的稳定性；
2. 接口文档清晰；
3. 可通过 Docker 快速启动；
4. 项目具备更强工程化展示价值。

## 第三阶段：前端页面与可视化

周期：第 4 周

目标：完成基础前端页面，让项目可以演示。

主要任务：

1. 登录注册页面；
2. 简历上传页面；
3. 岗位管理页面；
4. AI 分析报告页面；
5. 投递记录页面；
6. 个人数据看板页面。

阶段成果：

1. 用户可以通过浏览器完整体验系统；
2. 项目可以录制演示视频；
3. GitHub README 可以展示页面截图。

## 第四阶段：高级功能扩展

周期：后续迭代

目标：增加 AI 应用亮点。

可选任务：

1. WebSocket 实时返回 AI 分析进度；
2. 面试问题自动生成；
3. 简历版本对比；
4. 岗位推荐；
5. RAG 岗位知识库；
6. 多模型切换；
7. 管理员后台。

## 7. 项目验收标准

项目第一版完成时，应满足以下标准：

1. 后端服务可以正常启动；
2. 用户可以完成注册和登录；
3. JWT 鉴权有效；
4. 用户可以上传简历；
5. 系统可以解析简历文本；
6. 用户可以录入岗位 JD；
7. 系统可以调用 AI 生成匹配分析；
8. 用户可以保存和管理投递记录；
9. Swagger 接口文档完整；
10. GitHub README 清晰；
11. 项目能够写入简历并用于面试讲解。
