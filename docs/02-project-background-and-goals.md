# InternPilot 项目背景与目标

## 1. 项目背景

随着互联网、人工智能和软件工程相关岗位竞争加剧，大学生在寻找暑期实习和日常实习时面临越来越高的简历筛选门槛。许多学生虽然具备一定课程项目、竞赛经历或个人项目经历，但在简历表达、岗位匹配、投递管理和面试准备方面缺乏系统方法。

传统求职过程中，学生通常需要手动整理简历、复制岗位 JD、凭个人经验判断岗位匹配度，并使用表格或备忘录记录投递状态。这种方式存在效率低、信息分散、反馈不及时、简历优化缺乏针对性等问题。

近年来，大语言模型在文本理解、信息抽取、内容生成和语义分析方面表现出较强能力。将大语言模型应用到实习求职场景，可以帮助学生更好地理解岗位要求、分析自身短板、优化简历内容，并形成系统化的投递管理流程。

因此，本项目计划构建 InternPilot，一个面向大学生的 AI 实习投递与简历优化平台。

## 2. 项目目标

InternPilot 的目标是构建一个集简历管理、岗位管理、AI 分析和投递跟踪于一体的实习求职辅助平台。

具体目标包括：

1. 提供用户注册登录与个人信息管理能力；
2. 支持用户上传 PDF/DOCX 格式简历，并解析简历文本；
3. 支持用户录入和管理多个实习岗位 JD；
4. 基于 AI 模型分析简历与岗位之间的匹配度；
5. 输出简历优势、短板、技能缺口和优化建议；
6. 支持用户管理实习投递记录和面试状态；
7. 通过 Redis 缓存减少重复 AI 调用；
8. 提供清晰的接口文档和可部署的工程结构；
9. 最终形成一个适合写入简历、展示 GitHub 和面试讲解的完整项目。

## 3. 目标用户

本项目主要面向以下用户：

1. 正在寻找暑期实习的大学生；
2. 准备投递 Java 后端、前端、AI 应用等岗位的学生；
3. 缺少简历优化经验的低年级本科生；
4. 希望系统管理投递流程的求职者。

## 4. 核心业务流程

用户使用流程如下：

1. 用户注册并登录系统；
2. 上传自己的简历文件；
3. 系统解析简历文本；
4. 用户录入目标岗位 JD；
5. 系统调用 AI 模型进行匹配分析；
6. 用户查看匹配分数和优化建议；
7. 用户保存投递记录；
8. 用户持续更新投递状态和面试记录。

## 5. 项目核心价值

### 5.1 对用户的价值

1. 降低简历优化门槛；
2. 提高岗位理解效率；
3. 帮助用户发现技能短板；
4. 提升投递管理效率；
5. 辅助面试准备。

### 5.2 对开发者的价值

1. 展示 Spring Boot 后端工程能力；
2. 展示 Spring Security + JWT 权限认证能力；
3. 展示文件上传、解析和存储能力；
4. 展示 Redis 缓存优化能力；
5. 展示 AI 应用集成能力；
6. 展示完整项目规划、数据库设计、接口设计和部署能力。

## 6. 项目边界

第一阶段重点完成 MVP，不追求功能过多。

### 第一阶段包含

1. 用户认证；
2. 简历上传与解析；
3. 岗位 JD 管理；
4. AI 匹配分析；
5. 投递记录管理；
6. Swagger 接口文档。

### 第一阶段不包含

1. 自动投递；
2. 爬取招聘网站；
3. 在线编辑完整简历模板；
4. 复杂 RAG 知识库；
5. 多用户社交功能；
6. 商业化支付系统。

## 7. 预期成果

项目完成后应具备以下成果：

1. 一个完整的 GitHub 仓库；
2. 一套清晰的项目文档；
3. 一个可运行的 Spring Boot 后端服务；
4. 一个基础可用的前端页面；
5. 一套完整的数据库表结构；
6. 可访问的 Swagger 接口文档；
7. 可写入简历的项目经历描述；
8. 可用于面试讲解的技术总结。
