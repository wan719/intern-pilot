# InternPilot 产品体验验收与 Bug 收敛设计文档

## 一、文档目的

本文档用于指导 InternPilot 项目进入“产品体验验收 + Bug 收敛 + 演示稳定化”阶段。

当前项目已经完成多个核心模块：

1. 用户认证与 JWT 鉴权
2. RBAC 权限系统
3. 简历上传与解析
4. 岗位 JD 管理
5. AI 简历匹配分析
6. WebSocket AI 分析进度推送
7. Redis 任务状态保存与刷新恢复
8. AI 分析缓存与 Mock AI 多场景增强
9. DeepSeek 真实 API 接入
10. AI 面试题生成增强
11. RAG 岗位知识库
12. 系统操作日志
13. 管理员后台
14. 本地开发与 Docker 部署支持

当前阶段的重点不是继续增加新功能，而是对已有功能进行系统化验收，修复影响演示和使用体验的问题，使项目达到可以稳定展示、可以写进 README、可以用于实习投递和面试讲解的状态。

---

## 二、当前阶段定位

### 2.1 当前项目状态

项目已经从早期功能开发阶段进入体验收敛阶段。

```text
前一阶段：功能补全
当前阶段：体验验收、Bug 收敛、演示稳定化
后一阶段：README 包装、演示视频、服务器部署
````

当前不应继续盲目增加新功能，否则会导致：

1. 功能越来越多，但主流程不稳定
    
2. Bug 越积越多
    
3. 前端体验不统一
    
4. 面试演示时容易翻车
    
5. Git 提交和分支历史混乱
    

---

### 2.2 本阶段核心目标

本阶段目标是：

> 对 InternPilot 已有功能做完整体验验收，集中修复 P0 / P1 问题，确保项目主流程稳定、错误提示清晰、权限边界正确、AI 功能可演示、前后端构建和测试通过。

---

## 三、本阶段不做什么

本阶段暂时不做以下内容：

1. 不新增大型业务模块
    
2. 不继续扩展 RBAC 高级功能
    
3. 不继续扩展 RAG 高级检索
    
4. 不新增面试题收藏和刷题记录
    
5. 不新增 AI 评分和多轮模拟面试
    
6. 不新增复杂数据看板
    
7. 不做大规模 UI 重构
    
8. 不做服务器正式部署
    
9. 不重构整体架构
    
10. 不删除 Mock AI
    

本阶段只做：

```text
验收已有功能
修复 P0/P1 Bug
统一体验
补充必要测试
整理演示链路
```

## 四、Bug 分级标准

### 4.1 P0：必须立即修复

P0 是阻塞主流程的问题。

示例：

|类型|示例|
|---|---|
|登录失败|admin / demo 无法登录|
|接口崩溃|正常操作返回 500|
|主流程断裂|简历上传后无法分析|
|AI 功能不可用|AI 匹配分析失败|
|删除失败|分析报告、面试题无法删除|
|权限绕过|普通用户能访问后台接口|
|页面白屏|刷新后前端崩溃|
|测试失败|后端 test 或前端 build 不通过|

P0 必须优先修复，不能带着 P0 合并到 `dev`。

---

### 4.2 P1：优先修复

P1 是影响演示体验的问题。

示例：

|类型|示例|
|---|---|
|输出语言问题|AI 返回英文|
|错误提示差|只显示 unknown error|
|状态不刷新|删除后列表不更新|
|Loading 缺失|点击生成后无反馈|
|空数据体验差|空表格没有提示|
|刷新状态丢失|页面刷新后任务状态消失|
|按钮权限异常|无权限按钮仍显示|
|表单校验缺失|不选简历也能提交|

P1 应在本阶段尽量修复。

---

### 4.3 P2：后续优化

P2 不影响主流程，但影响细节体验。

示例：

|类型|示例|
|---|---|
|样式不美观|表格列宽不合理|
|页面布局一般|卡片间距不统一|
|chunk 体积提示|Vite 大 chunk warning|
|交互动效缺失|没有过渡动画|
|文案不够精致|页面提示语不够自然|

P2 可以记录，但本阶段不强制全部修复。

---

### 4.4 P3：长期规划

P3 属于未来增强。

示例：

1. 多轮模拟面试
    
2. AI 评分
    
3. 面试题收藏
    
4. 刷题记录
    
5. 在线部署
    
6. 自动化演示脚本
    
7. 性能压测
    
8. 前端分包优化

## 五、验收主流程

本阶段必须优先保证以下完整主流程稳定。

### 5.1 普通用户主流程

```text
1. demo 登录
2. 查看首页
3. 上传或选择简历
4. 查看简历列表
5. 查看简历版本
6. 新建岗位 JD
7. 查看岗位列表
8. 发起 AI 匹配分析
9. 查看 WebSocket 分析进度
10. 查看 AI 分析报告详情
11. 删除 AI 分析报告
12. 重新生成 AI 分析报告
13. 基于报告生成面试题
14. 查看面试题列表
15. 查看面试题详情
16. 重新生成面试题
17. 删除面试题
18. 刷新页面，确认登录态和页面状态正常
```

---

### 5.2 管理员主流程

```text
1. admin 登录
2. 进入管理员后台
3. 查看用户管理
4. 查看角色管理
5. 查看权限管理
6. 查看 RAG 知识库
7. 新增 / 编辑 / 删除 RAG 知识
8. 查看系统操作日志
9. 检查普通用户无法访问后台
10. 刷新后台页面，确认不会误跳 403
```

---

### 5.3 AI 主流程

```text
1. 确认 ai.provider=deepseek 时走 DeepSeekAiClient
2. 确认 ai.provider=mock 时走 MockAiClient
3. 不同简历 / 岗位分析结果不应错误复用
4. 同一简历 / 岗位命中缓存时应有合理表现
5. DeepSeek 输出应为简体中文
6. AI 返回非法 JSON 时应有明确提示
7. AI 分析失败时前端不白屏
8. WebSocket 进度正常推进
9. 页面刷新后可恢复任务状态
```

## 十一、接口验收重点

### 11.1 不允许出现的情况

1. 正常操作返回 500
    
2. 权限不足返回 500
    
3. 删除不存在数据返回 500
    
4. AI 解析失败导致页面白屏
    
5. 业务异常无提示
    
6. 后端日志打印 API Key
    
7. MockAiClient 被删除
    
8. 测试依赖真实 DeepSeek API
    

---

### 11.2 推荐错误响应

业务异常建议返回统一结构：

```json
{
  "code": "AI_RESPONSE_PARSE_FAILED",
  "message": "AI 返回格式异常，请稍后重试"
}
```

权限异常：

```json
{
  "code": "FORBIDDEN",
  "message": "没有权限访问该资源"
}
```

未登录：

```json
{
  "code": "UNAUTHORIZED",
  "message": "请先登录"
}
```

|检查项|要求|
|---|---|
|API Key|只来自环境变量|
|日志|不输出 API Key|
|默认模型|deepseek-v4-flash|
|Pro 模型|deepseek-v4-pro|
|中文输出|必须简体中文|
|JSON 输出|结构化场景必须合法 JSON|
|失败处理|有明确错误码|
|测试|不真实请求 DeepSeek|
